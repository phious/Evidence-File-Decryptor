Encryption Scheme
Algorithm
AES-256-CBC (mbedTLS, firmware v12.0)
Key Size
256-bit (32 bytes), set via BLE CHR_AESKEY or hardcoded in firmware
IV Derivation
First 16 bytes of DEVICE_ID, zero-padded. EMERGENCY_DEVIC → 454d455247454e43595f44455649430000
Chunk Size
1024 bytes per chunk, padded to 16-byte boundary. IV chains across all chunks (single CBC stream).
Padding
PKCS#7 compatible. Pad byte = number of padding bytes. Final chunk only.
Audio Format
WAV, 8-bit mono. Nominal 8kHz, actual ~9524 Hz. 30s = ~286k bytes at true rate.
Hash
SHA-256 of the .enc file written to events.txt. Use Verify tab to confirm integrity.
Firmware Default Key

Hardcoded in firmware source. If you changed it via BLE, use your custom key instead.
